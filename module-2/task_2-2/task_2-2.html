1. Where should you use constants, and where should you use variables? Can they all be constants?

    2. Constants should be used for values that do not change throughout the program, while variables should be used for values that may change. Not all values can be constants, as some need to be updated or modified.

  2. How do parentheses affect the arithmetic outcome of an expression?

    3. Parentheses are used to group expressions and control the order of operations. They ensure that the operations within the parentheses are performed first, which can change the outcome of the expression.

  3. How do we limit floating-point numbers to a given number of decimals?

    4. We can limit floating-point numbers to a specific number of decimals using methods like toFixed() in JavaScript, which formats a number using fixed-point notation.

  4. How can we compare two floating-point values to see if they are equal (is 12.199999 the same as 12.2)?

    5. To compare two floating-point values for equality, we can use a small tolerance value (epsilon) to check if the difference between the two values is within an acceptable range. For example: ```javascript const epsilon = 0.00001; const isEqual = Math.abs(12.199999 - 12.2) < epsilon; ```

  5. What do comparison operators return, and how can we check if they are FALSE or TRUE?

    6. Comparison operators return boolean values: TRUE or FALSE. We can check their values directly in conditional statements or by logging them to the console.

  6. How does the Math.random() function work, and what can it be used for?

    7. Math.random() generates a random floating-point number between 0 (inclusive) and 1 (exclusive). It can be used for various purposes, such as creating random values, simulating randomness in games, or selecting random elements from an array.

  7. What is modulus (%), and when do we need this operator in mathematical operations?

    8. The modulus operator (%) returns the remainder of a division operation. It is useful for determining if a number is even or odd, cycling through a range of values, or implementing wrap-around behavior in calculations.
