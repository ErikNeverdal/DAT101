1. What is the difference between invoking and creating a function?

    2. Creating a function means defining it, while invoking a function means executing it.

  2. Explain the syntax regarding semicolons and code blocks.

    3. In JavaScript, semicolons are used to terminate statements. Code blocks are defined using curly braces { } and are used to group statements.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. When a function doesn't return a value, it implicitly returns undefined. If we try to store the result of such a function, the variable will hold the value undefined.

  4. How do we separate parameters from each other when creating a function?

    5. We separate parameters from each other using commas. For example, in the function definition function myFunction(param1, param2), the parameters param1 and param2 are separated by a comma.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. During the creation of a function, we define parameter names that act as placeholders for the values that will be passed in when the function is invoked. When invoking the function, we provide arguments that are matched to the parameter names in the order they are defined.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. If a function is called without providing a value for a parameter, that parameter is assigned the value undefined. This is because JavaScript does not throw an error for missing arguments; instead, it initializes them to undefined.

  7. Explain what happens when we use recursive functions.

    8. When we use recursive functions, the function calls itself with modified arguments in order to solve a problem. Each recursive call creates a new execution context, and the function continues to call itself until it reaches a base case that stops the recursion.
